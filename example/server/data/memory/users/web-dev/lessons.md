- Always use TypeScript for better type safety
- Prefer functional components over class components in React
- Use environment variables for sensitive configuration
- Write tests for critical business logic
- Keep components small and focused
- Use proper error handling and logging
- Add user-friendly error messages for server connectivity issues
- Support project-level config overrides (e.g., .opencode/agent-memory.json) for per-project settings
- Use commands (/capture) for user-triggered session actions rather than tools
